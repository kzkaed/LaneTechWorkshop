HTML : Hyper Text Markup Protocol

1 : HTML

HTML is a markup language for describing web documents (web pages). HTML stands for Hyper Text Markup Language. A markup language is a set of markup tags. HTML documents are described by HTML tags. Each HTML tag describes different document content.

2: DOM

The Document Object Model is a programming interface for HTML, XML and SVG documents. It provides a structured representation of the document (a tree) and it defines a way that the structure can be accessed from programs so that they can change the document structure, style and content.

3: tag

The tag is label wraps the content for the page.

<body> and </body> are tags.

<div> and </div> are tags.

4: attribute

Attributes are added to tags to describe more details about the element.

<div class="circle">

class="circle" is an attribute of the div tag.

5: element

<div class="circle">Hi, I am a circle!</div> is an element. An element is the tag, attribute and the content inside the tags. An element is a node in the DOM. What is contained between the tags is what is displayed when the HTML is parsed by our browser. It is what is seen on the webpage.

html, head and body tags

A base HTML DOM has these core tags. —>

Notice the 'Tree' structure, where the head tag and body tag are contained inside the top level html tag. As you add more tags to the document, the structure becomes more nested, like branches of a tree. As a tree trunk moves up, branches extend from the trunk and branches from the branches before.

These are also called nodes. The parent node is the html tag. The child nodes are the head and body tags.

What goes in the head tag

The head tage contains metadata - information about the data contained in the html document. It is not seen when looking at the web page, but it may link the page to other files that control the look, feel and logic of the page.

In this example, the html document is

  • linked to a 'stylesheet' file called 'workshop.css' of type 'text/css'.
  • and the javascript file (src) 'workshop.js' of type 'text/javscript' is included with the document.
  • async in the script element has to do with how/when the javascript file is parsed. See note below.

The script element allows for javascript code to be contained inside the script element. However, it is generally preferable for it to be in a separate file. A clear separation of logic from markup keeps code clean.
